----------
кейс создание счета
цель :создание нового счета

актер : юзер

предусловия: юзер залогинился и находиться на основной странице.

основной сценарий:

1 - юзер жмет кнопку добавления счета.
2- система отображает форму создания счета.
3- ю. заполняет необходимые поля(наименование, тип, валюта, остаток) и подтверждает.
4- система проверяет корректность заполнения.
5. - система создает счет.
6. -с. уведомляет ю. о создании счета.

альтернативные сценарии:
4а -  ошибки заполнения, система сообщает юзеру об ошибках, переход на шаг 3.
5б -ошибки при создании счета.
6б - система сообщает юзеру о проблемах при создании счета.

кейс корректировка остатка
цель :корректировка остатка на счете

актер : юзер

предусловия: юзер залогинился и находиться на основной странице.

5- юзер корректирует остаток и подтверждает.
6- система проверяет корректность заполнения.
7. - система корректирует счет.
8. -с. уведомляет ю. о коррекции счета.

альтернативные сценарии:
6а -  ошибки заполнения, система сообщает юзеру об ошибках, переход на шаг 4.
7б -ошибки при коррекции счета.
8б - система сообщает юзеру о проблемах при коррекции счета.

кейс удаление счета
цель удаление счета

актер : юзер

предусловия: юзер залогинился и находиться на основной странице.

основной сценарий:
1- юзер кликает на нужный счет в списке счетов(выбирает его)

7б - система сообщает юзеру о проблемах при удалении счета.

кейс корректировка дохода
цель :корректировка сделанной  ранее записи о доходе.

актер : юзер

предусловия: юзер залогинился и находиться на основной странице.

основной сценарий:
1- юзер нажимает кнопку отобразить список произведенных расходов
2 - система отображает список расходов
3 - юзер выбирает подлежащую корректировке запись (кликает по ней)
4- система подсвечивает выбранную запись.
5- юзер жмет на кнопку изменить запись.
6- система открывает форму с записью доступной для изменения.
7 - юзер вносит необходимые изменения, жмет сохранить.
8 - система проверяет корректность изменений.
9. - система изменяет запись
10 -система уведомляет юзера о коррекции записи о расходе.

альтернативные сценарии:
8а -  ошибки заполнения, система сообщает юзеру об ошибках, переход на шаг 6.
9б -ошибки при попытке внести изменения.
10б - система сообщает юзеру о проблемах при изменении записи.
